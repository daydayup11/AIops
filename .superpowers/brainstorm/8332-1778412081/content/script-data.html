Python 脚本怎么拿到数据？

脚本需要查询 ClickHouse，数据从哪里来？

A

脚本自己连 ClickHouse 执行 SQL（推荐）

Planner 生成的脚本包含完整的 SQL 查询逻辑，通过环境变量拿到连接参数后自行查询，再用 matplotlib/seaborn 等库生成图片。

优点

  • 脚本自包含，逻辑完整
  • LLM 可以在脚本里自由处理数据（过滤、聚合、pivot）
  • 不需要中间数据传递

缺点

  • SQL engineer 节点变成可选或合并进 planner
  • 脚本要处理连接异常
B

executor 先跑 SQL，把 DataFrame 序列化传给脚本

保留现有的 sql_engineer → executor 流程，executor 把查询结果以 CSV/JSON 文件形式写到临时目录，脚本读取文件做可视化。

优点

  • 保留现有 SQL 执行链路，改动更小
  • 脚本只做可视化，职责单一

缺点

  • 多一次数据序列化/反序列化
  • 脚本能做的数据处理受限（只能处理已查出来的数据）
