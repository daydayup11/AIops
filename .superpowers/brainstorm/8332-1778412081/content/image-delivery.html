图片如何传给前端？

脚本生成图片后，前端怎么拿到它？

A

Base64 内嵌在 WebSocket 消息里（推荐）

后端读取图片文件，base64 编码后直接塞进 WebSocket JSON 消息，前端用 <img src="data:image/png;base64,..."> 展示。

优点

  • 无需新增 HTTP 路由
  • 与现有 WebSocket 协议完全一致
  • 前端逻辑改动最小（新增一种 render 类型）

缺点

  • 大图会撑大 WS 消息（通常 <200KB，可接受）
B

静态文件 URL

FastAPI 挂载 /renders 静态目录，脚本保存图片后，后端推送 URL，前端用 <img src="/renders/..."> 加载。

优点

  • WS 消息轻量
  • 图片可缓存

缺点

  • 需要新增静态路由
  • 跨域/路径问题在前端需处理
  • 图片文件需要生命周期管理（清理）
