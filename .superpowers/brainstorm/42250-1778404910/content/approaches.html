实现方案对比

三种主题切换架构，选择最适合项目现状的方案

A

CSS 变量切换（推荐）

在 index.css 里定义两套 CSS 变量，给 <html> 加 data-theme="tech" 属性即可全局切换。React 中用一个 useTheme hook 管理状态并持久化到 localStorage。ECharts 注册自定义 theme JSON，在 theme 变化时传给 ReactECharts 的 theme prop。

优点

  • 改动最小，全量替换 CSS 变量即可
  • 无运行时计算，切换瞬间完成
  • 与现有 Tailwind CSS 变量体系完全兼容

缺点

  • 需要在 ECharts 上额外注册 theme JSON
B

Tailwind dark: 类切换

利用 Tailwind v4 的 dark: variant，在 <html> 上加/去 dark class。所有组件的暗色样式用 dark:bg-xxx 写在 JSX 里。

优点

  • 标准 Tailwind 方案，工具链支持好

缺点

  • 需要在每个组件里加大量 dark: 类，改动面广
  • 科技风不等于普通暗色，表现力受限
C

主题 Context + 动态 className

用 React Context 传递 theme 值，每个组件根据 theme 拼接不同 className 字符串。

优点

  • 每个组件可完全自定义两套样式

缺点

  • 改动量最大，每个组件都要改
  • 维护两套样式字符串容易出错

推荐 A：项目已经在用 CSS 变量体系（--color-background 等），加一套 tech 变量是最小改动路径，且与 Tailwind v4 的 @theme 完全兼容。
